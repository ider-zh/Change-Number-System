## ADDED Requirements

### Requirement: 管理员可以选中多条申请记录进行批量操作

系统 SHALL 在申请记录表格的每一行前显示复选框，允许管理员选中多条记录。表格头部提供全选/反选功能。选中状态 SHALL 仅保存在前端，不同步到 URL。

#### Scenario: 选中单条记录
- **WHEN** 管理员点击某行的复选框
- **THEN** 该行被选中，选中计数 +1

#### Scenario: 全选当前页记录
- **WHEN** 管理员点击表头的"全选"复选框
- **THEN** 当前页所有记录的复选框被选中，选中计数更新为当前页记录数

#### Scenario: 反选当前页记录
- **WHEN** 管理员在全选状态下再次点击表头"全选"复选框
- **THEN** 当前页所有记录的复选框被取消选中，选中计数归零

#### Scenario: 显示已选中数量
- **WHEN** 管理员选中 N 条记录
- **THEN** 表格顶部显示"已选择 N 项"

#### Scenario: 切换分页时清空选中状态
- **WHEN** 管理员切换到下一页
- **THEN** 当前选中状态被清空（因为不同页面的记录不同）

#### Scenario: 应用筛选时清空选中状态
- **WHEN** 管理员应用或修改筛选条件
- **THEN** 当前选中状态被清空

### Requirement: 管理员可以批量删除选中的申请记录

系统 SHALL 提供批量删除功能，管理员可以一次性删除多条选中的申请记录。批量删除 SHALL 需要二次确认以防止误操作。

#### Scenario: 显示批量删除按钮
- **WHEN** 管理员选中至少一条记录
- **THEN** 表格顶部浮动操作栏显示"批量删除"按钮

#### Scenario: 点击批量删除弹出确认
- **WHEN** 管理员点击"批量删除"按钮
- **THEN** 系统弹出确认对话框，显示"确定要删除选中的 N 条记录吗？此操作不可撤销。"

#### Scenario: 确认批量删除
- **WHEN** 管理员在确认对话框中点击"确定"
- **THEN** 系统调用 `DELETE /api/applications/batch` 并传递选中的 ID 数组

#### Scenario: 批量删除成功
- **WHEN** 批量删除 API 返回成功
- **THEN** 系统显示成功提示"已删除 N 条记录"，刷新表格数据，清空选中状态

#### Scenario: 批量删除部分失败
- **WHEN** 批量删除 API 返回部分成功（如 10 条中删除了 8 条，2 条不存在）
- **THEN** 系统显示提示"成功删除 8 条，2 条记录不存在"，刷新表格数据

#### Scenario: 批量删除完全失败
- **WHEN** 批量删除 API 返回错误（如权限不足）
- **THEN** 系统显示错误提示，不清空选中状态

#### Scenario: 取消批量删除
- **WHEN** 管理员在确认对话框中点击"取消"
- **THEN** 对话框关闭，选中状态保持不变

### Requirement: 批量删除后端接口

系统 SHALL 提供批量删除接口，接收 ID 数组并删除对应的记录。

#### Scenario: 批量删除存在的记录
- **WHEN** 管理员调用 `DELETE /api/applications/batch` 并提供 `[1, 2, 3]`
- **THEN** 系统删除所有指定记录并返回 `{ deleted: 3 }`

#### Scenario: 批量删除包含不存在的 ID
- **WHEN** 管理员调用批量删除接口，ID 2 不存在
- **THEN** 系统删除 ID 1 和 3，返回 `{ deleted: 2, notFound: 1 }`

#### Scenario: 批量删除空数组
- **WHEN** 管理员调用批量删除接口并提供空数组 `[]`
- **THEN** 系统返回 400 错误并提示"请提供要删除的记录 ID"

#### Scenario: 非管理员调用批量删除
- **WHEN** 普通用户调用批量删除接口
- **THEN** 系统返回 403 禁止访问错误
