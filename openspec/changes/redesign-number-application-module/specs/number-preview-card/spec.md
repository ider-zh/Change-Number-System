## ADDED Requirements

### Requirement: 编号预览卡片展示编号格式

系统 SHALL 在选择完项目代号和编号类型后展示编号预览卡片，显示即将生成的编号格式。

#### Scenario: 预览卡片在选择完成后出现
- **WHEN** 用户已在 Wizard 中选择了项目代号和编号类型
- **THEN** 系统在确认步骤展示编号预览卡片

#### Scenario: 预览卡片展示编号格式模板
- **WHEN** 预览卡片渲染
- **THEN** 系统展示格式为 `{项目代号}-{编号类型}-{序号}` 的预览，其中序号部分显示为 `???` 或预估下一个序号

#### Scenario: 预览卡片标注为格式预览
- **WHEN** 预览卡片渲染
- **THEN** 系统在卡片显著位置标注"格式预览"或类似文字，明确告知用户最终编号以提交结果为准

### Requirement: 预览卡片展示已选信息摘要

系统 SHALL 在预览卡片中同时展示用户已选择的项目和编号类型信息。

#### Scenario: 展示项目名称和编号类型名称
- **WHEN** 预览卡片渲染
- **THEN** 系统展示完整的项目代号及名称（如 `ALPHA01 - Alpha项目`）和编号类型代号及名称（如 `CR - 变更请求`）

#### Scenario: 待审核状态标识
- **WHEN** 用户选择的项目或编号类型状态为 `pending`
- **THEN** 预览卡片在对应字段旁展示"待审核"标识

### Requirement: 预览卡片支持不同步骤间的可见性

系统 SHALL 控制预览卡片仅在确认步骤（第四步）展示，不在此前步骤中出现。

#### Scenario: 前序步骤不展示预览卡片
- **WHEN** 用户处于步骤一、二、三
- **THEN** 系统不渲染编号预览卡片

#### Scenario: 确认步骤展示预览卡片
- **WHEN** 用户进入步骤四（确认提交）
- **THEN** 系统渲染编号预览卡片，位于提交按钮上方
